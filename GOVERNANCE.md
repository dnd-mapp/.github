# Governance: D&D Mapp

## 1. Project Philosophy

D&D Mapp is currently a personal project developed and maintained by **NoNamer777 (@NoNamer777)**. The project aims to provide high-quality tools for the TTRPG community.

While the source code is public, the project is not currently seeking external code contributions (Pull Requests) to ensure a consistent architectural vision and codebase.

## 2. Roles

### Project Lead (BDFL)

The Project Lead (NoNamer777) has total authority over the project, including:

- Setting the technical roadmap and feature priorities.
- Final say on all architectural decisions.
- Managing repository settings, releases, and documentation.

### Community Members

Users and enthusiasts of D&D Mapp are encouraged to participate in the project's evolution through feedback.

## 3. How to Participate

Although this is a personal project, community input is highly valued. You can contribute by:

- **Bug Reports:** Open a GitHub Issue if you find something broken. Please provide steps to reproduce.
- **Feature Requests:** Suggest new ideas via GitHub Issues or Discussions. Use the "Enhancement" label if available.
- **Questions/Support:** Use the GitHub Discussions tab for general questions about how to use the tools.
- **Feedback:** Share how you are using the app! This helps prioritize which features to build next.

## 4. Decision-Making Process

Decisions are made at the discretion of the Project Lead.

1. **Input:** Ideas are gathered from Issues and Discussions.
2. **Evaluation:** The Project Lead evaluates suggestions based on technical feasibility, project goals, and personal bandwidth.
3. **Execution:** Accepted ideas are added to the internal roadmap and implemented by the Project Lead.

## 5. Future Changes

As the project grows, this governance model may evolve. If the project reaches a point where it is ready to accept external maintainers or contributors, this document will be updated to reflect those changes.

## 6. Code of Conduct

All participants are expected to follow our [CODE_OF_CONDUCT.md](https://github.com/dnd-mapp/.github?tab=coc-ov-file) to ensure a welcoming environment for all players and developers.

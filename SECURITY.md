# Security Policy

## Supported Versions

We actively provide security updates for the following versions of our projects. If you are running an older version, please upgrade to the latest stable release to ensure you have the most recent security patches.

## Reporting a Vulnerability

The D&D Mapp team takes the security of our software and our users' data seriously. If you believe you have found a security vulnerability, please report it to us privately using one of the methods below.

**Please do not report security vulnerabilities through public GitHub issues.**

### How to Report

1. **Private Vulnerability Reporting:** The preferred method is using GitHub's [Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidelines-for-reporting-a-vulnerability/reporting-a-vulnerability-to-a-repository) feature on the specific repository.
2. **Email:** Alternatively, you can send an email to [security@dndmapp.nl.eu.org](mailto:security@dndmapp.nl.eu.org). (Replace with your actual security contact email).

### What to Include

To help us prioritize and triage your report, please include:

- A description of the vulnerability.
- Steps to reproduce (proof of concept).
- Potential impact of the vulnerability.
- Any suggested mitigations.

## Our Response Process

After receiving a report, the D&D Mapp maintainers will:

1. Acknowledge receipt of the report within 48–72 hours.
2. Investigate the issue and confirm the vulnerability.
3. Work on a fix or mitigation.
4. Once a fix is ready, we will release a new version and, if applicable, publish a GitHub Security Advisory to credit the researcher and notify the community.

We request that you do not disclose the vulnerability publicly until we have had a reasonable amount of time to address it.

## Preferred Research Practices

- Avoid performing research that causes a Denial of Service (DoS) to our production services.
- Do not attempt to access, modify, or delete data belonging to other users.
- Use your own accounts/campaigns for testing purposes.

# Support Guide

First of all, thank you for using D&D Mapp! We want to ensure you have the best experience possible while building your worlds and running your campaigns.

This document outlines the best ways to get help, report issues, and stay up-to-date with the project.

## 🛠️ How to Get Help

### 1. Check the Documentation

Before opening an issue, please check our official documentation (if applicable) and the README of the specific repository you are using. Most configuration and setup questions are covered there.

### 2. GitHub Discussions

For general questions, "How-to" inquiries, or sharing your maps and ideas, please use the **Discussions** tab in the relevant repository. This helps keep the issue tracker focused on actionable bugs and features.

- [Browse Discussions](https://github.com/orgs/dnd-mapp/discussions)

### 3. Reporting Issues

If you believe you've found a bug, please open an issue in the specific repository where the problem occurs.

When filing an issue, please include:

- A clear, descriptive title.
- Steps to reproduce the behavior.
- Expected vs. actual behavior.
- Screenshots or logs (if applicable).
- Your environment details (OS, Browser, Node version, etc.).

[Open an Issue](https://github.com/dnd-mapp)

### 4. Feature Requests

We love hearing new ideas for D&D Mapp! Please search existing issues and discussions first to see if your idea has already been suggested. If not, open a new issue using the "Feature Request" template.

## 🛡️ Security Policy

If you discover a security vulnerability, please **do not** open a public issue. Instead, please email us at [security@dndmapp.nl.eu.org](mailto:security@dndmapp.nl.eu.org) or use the GitHub "Report a vulnerability" feature if enabled.

## 🕒 Response Times

D&D Mapp is maintained by community contributors and volunteers. While we try to respond to issues and pull requests promptly, please understand that response times may vary.
